# Site V1.3 Complete Technical SEO Design

## Goal

Make every public page directly crawlable without JavaScript, with China-first search discovery and standards-compatible metadata for Baidu, Google, and Bing, while preserving the existing React + Vite architecture and production behavior.

## Search Priorities

- Primary: Baidu and other Chinese search crawlers that may not execute client JavaScript reliably.
- Secondary: Google and Bing, including canonical URL, sitemap, structured data, and social metadata compatibility.
- The public site remains Chinese-only, so the document language is `zh-CN`; no hreflang matrix is required.

## Build-Time Rendering Architecture

The client application keeps its current `BrowserRouter`, route-level lazy loading, and performance budget. V1.3 adds a separate Vite SSR entry used only during the build. That entry uses `StaticRouter`, synchronous page imports, the existing page components, `MainLayout`, and `react-dom/server` to produce the same visible React body for a requested route.

The production build sequence will:

1. build the existing client bundle;
2. build the temporary server renderer into a dedicated build directory;
3. enumerate every public static, news, and project route;
4. render each route body and inject it into the client HTML template;
5. inject route-specific metadata, canonical links, and JSON-LD;
6. generate sitemap and crawler files;
7. remove temporary server-rendering output;
8. run SEO validation and the existing bundle budget.

The browser continues to mount the existing client application. No server runtime, database, framework migration, or request-time rendering is introduced.

## Route Source of Truth

A build-time route manifest will describe static routes and discover published news/project routes from Markdown. The prerenderer, sitemap generator, and validation scripts will consume this manifest so their route sets cannot drift.

Only content with `publishStatus: published` is included in the sitemap and static public route set. Draft content remains excluded.

## Static HTML Requirements

Every indexed route must contain before JavaScript runs:

- a non-empty React-rendered body inside `#root`;
- exactly one meaningful page `h1`;
- its visible primary copy or article/project content;
- route-specific title and description;
- one absolute canonical URL using the established trailing-slash convention;
- Open Graph and Twitter metadata;
- route-appropriate JSON-LD.

The generated body must not contain only a suspense loading fallback.

## Structured Data

Every page includes a shared JSON-LD graph containing `Organization` and `WebSite`. Route-specific nodes are added as follows:

- static pages: `WebPage`;
- news detail: `Article`, including headline, description, image, published date, author/publisher, and canonical URL;
- project detail: `CreativeWork`, including name, description, image, creator, and canonical URL;
- non-home routes: `BreadcrumbList` using Chinese labels and canonical URLs.

Runtime metadata managed by `SEOMeta` will include canonical and equivalent JSON-LD so client-side navigation does not leave stale head data.

## Sitemap, Robots, and Errors

- Generate a valid UTF-8 `sitemap.xml` with absolute canonical URLs for every indexed route.
- Generate `robots.txt` with `User-agent: *`, `Allow: /`, and `Sitemap: https://lekeopen.com/sitemap.xml`.
- Preserve Cloudflare content-signal policy where possible, but production acceptance must confirm the final served robots file still allows search and advertises the sitemap. If Cloudflare managed robots overrides the deployed file, the external dashboard setting must be corrected before V1.3 is closed.
- Add a static `404.html` with `noindex, nofollow`, Chinese copy, and a home link so unknown routes return a real not-found document instead of an indexable SPA shell.

Search-engine ownership verification tokens are optional deployment configuration. They are not embedded until the user supplies official Baidu, Google, or Bing verification values.

## Language and URL Policy

- Change the root document to `<html lang="zh-CN">`.
- Canonical URLs use HTTPS, `lekeopen.com`, and a trailing slash except file endpoints such as `sitemap.xml` and `rss.xml`.
- Sitemap, Open Graph URLs, runtime canonical links, and prerendered canonical links must use the same normalizer.

## Quality Gates

Automated tests and build validation will verify:

- the root language;
- canonical and JSON-LD output;
- sitemap XML validity, absolute URLs, uniqueness, and route coverage;
- published-only content inclusion;
- robots sitemap discovery;
- real body and `h1` content in generated route HTML;
- no suspense-only prerender output;
- 404 noindex behavior;
- all existing tests, TypeScript, ESLint, build, prerender, and bundle budget remain green.

Production acceptance will fetch representative static, news, and project pages with JavaScript disabled semantics (raw HTML) and confirm body copy, canonical, JSON-LD, sitemap, robots, RSS, and HTTP status behavior.

## Rollback

V1.3 changes only build tooling, head metadata, generated static output, and documentation. It has no content schema, database, or API migration. Reverting the V1.3 commits restores the previous meta-only prerender flow without affecting Markdown content or client routes.

## Out of Scope

- Framework migration to Next.js, Astro, or another site generator.
- Server-side runtime or edge rendering.
- Search ranking guarantees, paid search, keyword campaigns, or backlink work.
- Automatic submission to webmaster accounts without user-owned verification credentials.
- Additional languages or hreflang support.
