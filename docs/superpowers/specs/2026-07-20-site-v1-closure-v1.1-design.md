# 官网 V1.0 收尾与 V1.1 工程质量设计

## 目标

关闭已完成的 V1.0 产品范围，并为 V1.1 建立不扩张业务功能的工程质量计划。

## V1.0 范围决策

- 客户反馈模块从 V1.0 取消，官网不得编造或发布未经核实的客户评价；
- 联系页增加高德地图跳转，保留文字地址，不嵌入地图 SDK，不申请 API Key；
- 原始骨架 PRD 更新为逐项验收记录；
- 测试、类型检查、Lint 和生产构建全部通过后，V1.0 才能标记完成。

## V1.1 架构边界

V1.1 只建设发布质量门禁：统一验证命令、Markdown frontmatter 校验、GitHub Actions 和文档同步。保持现有 React、Vite、Markdown、EmailJS 和静态部署架构，不增加运行时服务或依赖。

## 验证策略

- 使用 Node 内置测试运行器，避免引入测试框架；
- 地图入口测试检查高德 URI、外部链接安全属性和可见文案；
- 内容校验测试使用临时 Markdown 样本覆盖有效内容与缺失字段；
- CI 直接调用与本地相同的 npm scripts；
- 完整生产构建作为最终发布门禁。

## 风险控制

- 外部地图只跳转，不向页面引入第三方脚本；
- 构建生成的 RSS 和发布队列需区分真实内容变化与时间戳变化；
- 当前主 JavaScript 包较大，V1.1 先记录基线，只有测量结果支持时再拆包；
- EmailJS 的 `VITE_*` 配置是公开客户端标识，不得误放服务端秘密。
