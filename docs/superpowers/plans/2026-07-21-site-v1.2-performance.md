# Site V1.2 Performance and Observability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce the initial JavaScript entry chunk by at least 30%, meet the approved mobile LCP and CLS targets, and add deterministic performance regression protection.

**Architecture:** Preserve the existing router and layout while moving page modules behind `React.lazy` boundaries rendered inside a stable layout-level `Suspense` fallback. Measure the initial module script referenced by `dist/index.html`, enforce a budget derived from the 999,991-byte V1.1 baseline, and keep timing-sensitive Lighthouse evidence outside the default CI gate.

**Tech Stack:** React 18, React Router 7, Vite 6, Node.js 24, Node.js test runner, Lighthouse

## Global Constraints

- Initial entry JavaScript must be no larger than 699,993 bytes, a 30% reduction from the 999,991-byte V1.1 baseline.
- Mobile laboratory LCP must be no greater than 2.5 seconds and CLS no greater than 0.1.
- Preserve existing routes, design, animation, SEO metadata prerendering, accessibility, RSS, map, and contact behavior.
- Do not add CMS, database, API, Playwright, or timing-sensitive Lighthouse checks to the default CI gate.
- Release through `develop` before `main`.
