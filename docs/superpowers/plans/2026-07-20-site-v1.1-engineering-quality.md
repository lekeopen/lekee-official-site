# Site V1.1 Engineering Quality Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a reproducible local and CI quality gate for code, Markdown content, and production builds without changing website features.

**Architecture:** Keep the static React and Vite runtime unchanged. Add a dependency-free Node content validator and tests, expose one npm verification command, then run the same command in GitHub Actions.

**Tech Stack:** Node.js 20, React 18, TypeScript 5.8, Vite 6, ESLint 9, Node built-in test runner, GitHub Actions.

## Global Constraints

- Do not add a CMS, database, API, or runtime service.
- Do not add a new npm dependency unless the existing parser cannot provide the required validation.
- Do not commit EmailJS credentials or any production secret.
- Preserve the `develop` to `main` release workflow.
- Keep generated RSS and publish queue changes only when content or the release intentionally changes them.

### Task 1: Validate Markdown frontmatter

**Files:**
- Create: `scripts/validate-content.mjs`
- Create: `tests/content-validation.test.mjs`
- Modify: `package.json`

**Interfaces:**
- Consumes: `content/news/*.md` and `content/projects/*.md`.
- Produces: `validateContent(rootDir): Promise<string[]>`; an empty array means valid content, otherwise each string identifies a file and invalid field.

- [ ] **Step 1: Write failing tests for valid and invalid content**

```js
test('reports missing required news fields', async () => {
  const errors = await validateContent(fixtureRoot('invalid-news'));
  assert.deepEqual(errors, ['content/news/missing-title.md: missing title']);
});

test('accepts valid news and project files', async () => {
  const errors = await validateContent(fixtureRoot('valid-content'));
  assert.deepEqual(errors, []);
});
```

- [ ] **Step 2: Run the focused test and confirm the missing module failure**

Run: `node --test tests/content-validation.test.mjs`

Expected: FAIL because `scripts/validate-content.mjs` does not exist.

- [ ] **Step 3: Implement deterministic validation**

Validate news fields `title`, `date`, `category`, `summary`, and `status`; allow only `draft` or `published`. Validate project fields `name`, `subtitle`, `status`, `publishStatus`, `summary`, `category`, and `tech_stack`; allow only the statuses declared by `ProjectItem` in `src/lib/content.ts`. Sort files and errors before returning them.

- [ ] **Step 4: Expose the content check**

Add this package script:

```json
"validate:content": "node scripts/validate-content.mjs"
```

- [ ] **Step 5: Verify the validator and full test suite**

Run: `npm run validate:content && npm test`

Expected: exit 0 with no content errors and all tests passing.

### Task 3: Run the same gate in GitHub Actions

**Files:**
- Create: `.github/workflows/quality.yml`

**Interfaces:**
- Consumes: `package-lock.json` and `npm run verify`.
- Produces: a `quality` workflow for pull requests and pushes to `develop` and `main`.

- [ ] **Step 1: Add the workflow**

```yaml
name: quality

on:
  pull_request:
  push:
    branches: [develop, main]

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run verify
```

- [ ] **Step 2: Validate workflow syntax and local parity**

Run: `npm ci && npm run verify`

Expected: exit 0 using only the lockfile-resolved dependency set.
