# V1.2 Performance Baseline

## V1.1 Production Build Baseline

Measured from commit `34aab55` with Node.js 24 and the production Vite build:

- Initial entry JavaScript: 999,991 bytes (`index-C90MvmDp.js`)
- Initial entry JavaScript gzip output: approximately 279,280 bytes
- Global CSS: approximately 46,730 bytes
- Total `dist/assets`: approximately 4.3 MB, including hidden source maps

The approved V1.2 entry budget is 699,993 bytes, exactly 30% below the V1.1 initial-entry baseline after rounding down to a whole byte.

## Measurement Commands

```bash
npm run build
npm run check:bundle
```

The bundle check reads the initial module script referenced by `dist/index.html`. It deliberately excludes source maps and route chunks because they are not part of the initial page request.

## V1.2 Acceptance Evidence

- Initial entry JavaScript after final route splitting: 230,080 bytes
- Reduction from V1.1 baseline: 77.0%
- Bundle budget result: passed
- Lighthouse version: 13.4.1, default mobile profile
- Test target: local Vite production preview in headless Chrome
- Performance score: 97
- Mobile FCP: 1,590 ms
- Mobile LCP: 2,369 ms
- Mobile CLS: 0.019
- Total blocking time: 0 ms

The first post-split run exposed a CLS regression caused by a 40vh route fallback. Reserving the remaining viewport reduced CLS from 0.522 to 0.019. Lighthouse then identified the animated hero heading as the LCP element; rendering that heading immediately reduced the final LCP to 2.369 seconds while preserving the animated background and supporting motion.
