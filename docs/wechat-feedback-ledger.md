# 公众号反馈台账

用途：把公众号、官网联系表单、微信私聊、线下沟通里的反馈统一记下来，便于周度回看和下篇选题。

原则：

- 先手工记录，不做自动化采集。
- 没有反馈也可以保留空表，方便后续补录。
- 只记录可行动的信息，不写未经核实的评价。

## 每周回看

每周只看三件事：

1. 哪些问题出现了 2 次以上。
2. 哪些场景可以写成下篇公众号内容。
3. 哪些问题需要补到产品待办或回复文案里。

## 建议填写方式

- `来源`：公众号留言、官网联系表单、微信私聊、电话、线下沟通。
- `问题类型`：体验问题、功能需求、价格咨询、合作咨询、内容建议。
- `处理状态`：待确认、已回复、已记录、已转待办、已写入文章。
- `下次动作`：补回复、补文案、补功能、写文章、继续观察。
