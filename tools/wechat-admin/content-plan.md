# 乐可开源服务号内容计划

定位：产品展示、信任背书、合作转化。

服务号不追求日更，重点是让用户快速理解：

- 我们是谁
- 我们有什么产品和能力
- 如何体验或合作

## 月度固定节奏

每月建议 2 篇主文，必要时增加 1 篇短更新。

### 第 1 篇：产品进展

目标读者：教育机构、老师、家长、潜在客户。

常见主题：

- GeoGenius 新知识点、新演示、新练习能力
- 乐教库资源管理、检索、审核、机构使用场景
- 小乐 AI 的智能体能力进展
- 产品真实使用反馈和小案例

标题方向：

- 我们做了一个帮小学生看懂几何的 AI 工具
- 老师的课件、教案和资源，为什么越存越难找？
- AI 教育产品不该只会聊天，它还应该帮学生看懂过程

### 第 2 篇：工程复盘

目标读者：企业客户、技术伙伴、开发者。

常见主题：

- OpenClaw / Clawdbot 部署实践
- AI Agent 如何进入真实工作流
- 官网、RSS、公众号运营工具建设
- 小团队如何做产品工程化和持续交付

标题方向：

- 我们把 OpenClaw 部署到云服务器后，AI 开始真正干活了
- 一个小团队如何维护官网、公众号和产品发布节奏
- 从官网 Markdown 到公众号草稿，我们为什么要做自己的运营工具

## 备用短更新

用于当月有新功能、小版本、活动或服务更新时发布。

- 产品小更新
- 菜单/入口更新
- 服务案例说明
- 开源项目进展
- 常见问题解答

## 每篇文章结构

1. 先讲真实问题，不要一上来宣布产品。
2. 说明我们怎么理解这个问题。
3. 展示产品、能力或工程实践。
4. 给出可体验入口或合作入口。
5. 引导关键词回复，例如“回复【几何】获取体验入口”。

## 反馈回流

每周同步一次公众号反馈台账：

- 记录来源、问题类型和原始反馈
- 标记是否高频
- 判断是否值得写进下一篇文章
- 如果本周没有反馈，就在台账里保留 `本周暂无反馈`
